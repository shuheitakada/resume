## 開発をする上で大切にしている考え方
- ビジネスの成功を見据えた開発をするように心がけています。別の言い方をすると、ビジネスの成功につながらない不必要な機能はなるべく開発しない方がいいと考えています。
- RailsにおいてはDB設計=モデル設計であり1つの設計がその後の開発に大きく影響を与えるので、DB設計はじっくり考えるようにしています。考え方としてはイミュータブルデータモデリングを大切にしています。
- 完璧主義にとらわれすぎてチームの開発生産性を落とすのは本末転倒なので、開発やレビュー時には80点付近を目指すように心がけています。
- コーディングをするときは無駄を省いてシンプルなコードを保つように心がけています。また、1つのファイル内やプロダクト内でなるべく統一性のある書き方をするようにしています。
- 個人よりもチーム全体の、また短期的ではなく中・長期的な開発生産性が上がるように開発を進めることが大事だと考えています。

## 仕事をする上で大切にしている考え方
- 仕事は個人で完結するものではなくチームで作り上げていくものだと考えているので、人とのコミュニケーションや雑談を大切にしています。
- 相手の仕事の妨げにならないように、基本的に即レスを心がけています。
- 自分の状況がチーム内の他のメンバーにも分かるように、報連相を大事にしています。
- 状況に応じて、チャットでの非同期コミュニケーションと通話での同期的なコミュニケーションを使い分けるようにしています。
